# ***2. src/task_planning***
`语义导航`
- 简而言之：这是一个“翻译官”节点，它听懂人话，查地图，然后告诉机器人：“先去哪，再去哪，如果找不到具体门牌号就先到大楼门口再慢慢找”
- `LLM` —— [ api_config.yaml](src/task_planning/config/api_config.yaml)
- [task_planning_sim.yaml](src/task_planning/config/task_planning_sim.yaml)
- [task_planning_server.py](src/task_planning/task_planning/task_planning_server.py)

```mermaid
flowchart TD

A[用户输入任务描述] --> B[生成 Prompt<br/>融合地图数据]
B --> C[调用 LLM API]

C --> D{返回结果是否为空}
D -- 否 --> E[清洗输出<br/>去除```等格式]
D -- 是 --> Z[报错返回]

E --> F{JSON是否合法}
F -- 否 --> Z
F -- 是 --> G[解析 delivery_plan]

G --> H[提取 building / unit / 坐标]

H --> I{是否有 unit 坐标}
I -- 有 --> J[生成 NAVIGATION 到 unit]
I -- 无 --> K[生成 NAVIGATION 到 building]
K --> L[追加 EXPLORATION 到 unit]

J --> M[加入任务序列]
L --> M

M --> N[返回 ROS2 TaskSequence]
```

# ***3. src/replanner***
核心功能：当机器人遇到障碍物时，动态优化局部路径，生成绕过障碍物的平滑路径
- `不重新规划整条路径，只优化“第二个点”`
- [waypoint_replanner.yaml](src/replanner/config/waypoint_replanner.yaml)
- [waypoint_replanner_server.py](src/replanner/replanner/waypoint_replanner_server.py)

```mermaid
flowchart TD
    A[接收重规划请求] --> B{路径有效?}
    B -->|否| Z[返回原路径]
    
    B -->|是| C[处理点云生成costmap]
    C --> D[获取机器人位姿]
    D --> E{路径点数≥2?}
    
    E -->|否| F[返回原路径]
    E -->|是| G[扇形采样优化第二个路径点]
    
    G --> H[计算代价函数<br/>障碍物+平滑性+位置]
    H --> I[选择最优点]
    
    I --> J[替换第二个点]
    J --> K[整体平移后续点]
    K --> L[返回优化路径]
    
    Z --> M([结束])
    F --> M
    L --> M
```

# ***5. src/grounded_sam2***
该模块基于 `Grounding DINO + SAM2`，实现`开放词汇目标检测`与`实例分割`，支持通过文本动态指定检测类别，并输出`像素级语义信息`，用于下游导航与环境理解

- [gsam2_params.yaml](src/grounded_sam2/config/gsam2_params.yaml)
- [gsam2_node.py](src/grounded_sam2/grounded_sam2/grounded_sam2/gsam2_node.py)

| 类型     | 说明           |
| ------: | ------------ |
| 分类     | 整张图一个标签      |
| 检测     | 框            |
| 语义分割   | 每像素类别（不区分个体） |
| 实例分割 | 每个物体 + 每像素   |


```mermaid
flowchart TD

A[订阅相机图像] --> B[缓存最新图像]

B --> C[定时器触发处理]

C --> D{是否有图像 & 未被占用}
D -- 否 --> C
D -- 是 --> E[图像预处理<br/>resize / sharpen]

E --> F[Grounding DINO检测]
F --> G[得到 boxes + labels + scores]

G --> H{是否检测到目标}
H -- 否 --> Z[结束]

H -- 是 --> I[坐标转换<br/>cxcywh → xyxy]

I --> J[SAM2分割]
J --> K[生成 masks]

K --> L[后处理]
L --> M[过滤小目标]
M --> N[生成 label_image]

N --> O[发布检测结果]
O --> P[发布可视化图像]
O --> Q[发布二值mask]
```

# ***9. src/delivery_bringup***
本系统采用分层规划架构，上层`基于OSRM进行全局路径规划`，下层`结合实时感知信息进行局部路径优化`，通过触发式replanning机制提升系统在动态环境中的鲁棒性。
- [delivery_bringup_sim.yaml](src/delivery_bringup/config/delivery_bringup_sim.yaml)
- src/delivery_bringup/launch/delivery_system_sim.launch.py

| 层级           | 谁在做               | 作用          |
| ------------ | ----------------- | ----------- |
| 🌍 全局规划      | OSRM              | 超远距离路径      |
| 🧠 局部规划（你写的） | WaypointReplanner | 避障 + 调整路径   |
| 🤖 控制执行      | Nav2              | 跟踪 waypoint |

```mermaid
flowchart TD
    A[OSRM 全局路径] --> B[Waypoint 插值]

    B --> C[Nav2 跟踪执行]

    C --> D{局部有障碍?}

    D -- 否 --> C

    D -- 是 --> E[调用你的 TriggerReplan]

    E --> F[点云生成局部costmap]

    F --> G[采样优化下一个 waypoint]

    G --> H[生成新路径（偏移）]

    H --> C
```
